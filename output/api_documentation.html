Dispensing Pharmacies API Documentation

Overview

This API routes prescription orders to the nearest pharmacy based on patient demographics and drug availability.

Entry Point

SearchController, /dispensingPharmacies/search, POST

Request

Field Type Description
NDC String National Drug Code of the drug
Patient ID String Unique identifier for the patient

Processing Flow

  1. Receive search request
  2. Fetch drug information using DrugAdaptor
  3. Retrieve member information using MembershipAdaptor
  4. Get pharmacy routing using RxRoutingAdaptor
  5. Apply business rules (GAC, QOH, DoD Override)
  6. Return optimal pharmacy and alternative pharmacies

External API Calls

API Name Purpose Called From
Drug API Fetch drug information DrugAdaptor

Business Rules

  • GAC Rule: Ensures that the pharmacy is capable of handling the drug.
  • QOH Rule: Verifies that the pharmacy has sufficient quantity on hand.
  • DoD Override: Overrides default rules based on doctor's order.

Response

Field Type Description
Optimal Pharmacy Pharmacy The pharmacy that can fulfill the prescription
Alternative Pharmacies List of Pharmacies Other pharmacies that can fulfill the prescription
