Dispensing Pharmacies API Documentation

Overview

This document outlines the API endpoints and business logic of the Dispensing Pharmacies microservice.

The service routes pharmacy requests based on various rules such as benefit type, geographic access criteria, quantity on hand, and military service branch validation.

Entry Point (Controller)

Endpoint: /dispensingPharmacies/search

HTTP Method: POST

Request (Fields)

Field NameTypeDescription
benefitTypeStringThe type of benefit, either "RETAIL" or "MAIL".

Processing Flow (Step by Step)

  1. Receive a POST request to /dispensingPharmacies/search with the SearchRequest payload.
  2. The SearchController forwards the request to the SearchService.
  3. The SearchService applies the following business rules in sequence:
    1. DoDOverrideRule: Applies if benefitType is "RETAIL". Returns a message or skips the rule.
    2. GACRule: Checks geographic access criteria. Always passes.
    3. QOHRule: Validates quantity on hand. Passes based on inventory levels.
    4. GetAllowedServiceBranchRule: Validates military service branch. Passes if valid.
  4. The SearchService selects the optimal pharmacy based on the rules and returns the result to the controller.
  5. The controller responds with the optimal pharmacy name as a string.
